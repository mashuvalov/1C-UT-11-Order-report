# Внешний отчет УТ 11: проданные товары в чеках и заказах

Отчет собирает уникальный список номенклатуры за выбранный период из проведенных документов:

- `Документ.ЧекККМ.Товары`;
- `Документ.ЗаказКлиента.Товары`.

Документы в результат не выводятся. Строки чеков и заказов объединяются, затем группируются по `Номенклатура`; количество суммируется. В отчет выводятся артикул, номенклатура, проданное количество, остаток с учетом заказов на `ДатаДо` и закупочная цена на `ДатаДо`.

## Файлы

- `МодульОбъекта.bsl` - регистрация внешнего отчета, поиск вида цены и основной запрос.
- `ФормаОтчета.bsl` - обработчики управляемой формы и вывод результата в табличный документ.

## Как собрать файл

1. Откройте базу УТ 11 в конфигураторе.
2. Создайте новую внешнюю обработку: `Файл` -> `Новый` -> `Внешняя обработка`.
3. Задайте имя обработки, например `ПроданныеТоварыЧекиЗаказы`.
4. Откройте модуль объекта внешней обработки и вставьте туда код из `МодульОбъекта.bsl`.
5. Создайте основную управляемую форму отчета, например `ФормаОтчета`.
6. На вкладке `Реквизиты` формы создайте два реквизита формы:
   - `ДатаОт`, тип `Дата`;
   - `ДатаДо`, тип `Дата`.
7. Перетащите реквизиты `ДатаОт` и `ДатаДо` на форму, чтобы появились два поля выбора даты.
8. На вкладке `Команды` формы создайте команду `Сформировать`.
9. Перетащите команду `Сформировать` на форму или на командную панель, чтобы появилась кнопка.
10. Откройте модуль формы и вставьте туда код из `ФормаОтчета.bsl`.
11. В свойствах формы откройте события и назначьте событие `ПриОткрытии` на процедуру `ПриОткрытии`.
12. У команды `Сформировать` назначьте действие на процедуру `Сформировать`.
13. Проверьте, что эта форма указана как основная форма внешней обработки.
14. Сохраните файл как `.epf`.
15. Запустите в режиме предприятия и откройте отчет через `Файл` -> `Открыть` или подключите его в `НСИ и администрирование` -> `Печатные формы, отчеты и обработки` -> `Дополнительные отчеты и обработки`.

## Настройки и важные места

- Период выбирается на форме в полях `ДатаОт` и `ДатаДо`.
- При первом открытии формы даты автоматически заполняются с начала текущего месяца по конец текущего месяца.
- После успешного формирования отчет запоминает выбранный период пользователя и при следующем открытии подставляет его.
- Закупочная цена берется из `РегистрСведений.ЦеныНоменклатуры.СрезПоследних` по выбранному виду цены.
- Вид цены на форме не выбирается. Код пробует найти вид цены с названием `Закупочная`, `Закупочные`, `Цена закупки` или `Закупочная цена`.
- Остаток берется из `РегистрНакопления.ТоварыНаСкладах.Остатки` на дату окончания периода, суммируется по всем складам и уменьшается на количество из проведенных заказов за период.
- Если заказ еще не реализован и товар физически числится на складе, отчет все равно покажет остаток за вычетом этого заказа.
- Если заказов больше, чем учетного остатка, в колонке `Остаток` будет 0.
- В коде есть две попытки для поля остатка: сначала `ВНаличииОстаток`, затем `КоличествоОстаток`. Это помогает, если в вашей редакции УТ 11 имя ресурса отличается.

## Если в вашей базе другие имена метаданных

В типовой УТ 11 обычно подходят эти имена:

- `Документ.ЧекККМ.Товары`;
- `Документ.ЗаказКлиента.Товары`;
- `РегистрНакопления.ТоварыНаСкладах`;
- `РегистрСведений.ЦеныНоменклатуры`;
- `Справочник.ВидыЦен`.

Если отчет при формировании выдаст ошибку запроса, откройте функцию `ПолучитьТекстЗапросаПроданныхТоваров` в `МодульОбъекта.bsl` и проверьте имя проблемного документа, регистра или поля по вашей конфигурации.

## Если при проверке формы пишет "Переменная не определена"

Ошибка вида `Переменная не определена (ДатаОт)` или `Переменная не определена (ДатаДо)` означает, что реквизиты периода не созданы в форме.

Проверьте форму:

- откройте вкладку `Реквизиты`;
- создайте `ДатаОт` и `ДатаДо` с типом `Дата`;
- убедитесь, что имена совпадают с кодом буква в букву;
- после этого снова выполните проверку модуля формы.

## Если форма открылась и ничего не произошло

Проверьте два пункта:

- в свойствах формы событие `ПриОткрытии` должно быть назначено на процедуру `ПриОткрытии`;
- форма с этим модулем должна быть основной формой внешнего отчета.

При правильной привязке при открытии поля `ДатаОт` и `ДатаДо` заполнятся текущим месяцем. Для формирования нажмите кнопку `Сформировать`.

## Если пишет "Метод объекта не обнаружен"

Ошибка вида `Метод объекта не обнаружен (НайтиЗакупочныйВидЦены)` обычно означает одно из двух:

- в модуль формы вставлена старая версия `ФормаОтчета.bsl`;
- в модуль объекта внешнего отчета не вставлен код из `МодульОбъекта.bsl`.

В актуальной форме вызов методов объекта идет через `РеквизитФормыВЗначение("Объект")`.

## Если при добавлении пишет "ХранилищеВариантов"

Ошибка `Поле объекта не обнаружено (ХранилищеВариантов)` появляется, когда файл создан как внешняя обработка, а регистрируется как дополнительный отчет.

В текущей версии `МодульОбъекта.bsl` стоит:

```bsl
ПараметрыРегистрации.Вставить("Вид", "ДополнительнаяОбработка");
```

Такой файл нужно добавлять в `Дополнительные отчеты и обработки` как дополнительную обработку. Пользователи смогут запускать его из интерфейса через список дополнительных обработок.

Если нужна именно регистрация как `ДополнительныйОтчет`, файл надо пересобрать как `Внешний отчет`, а не как `Внешняя обработка`, и вернуть в модуле объекта:

```bsl
ПараметрыРегистрации.Вставить("Вид", "ДополнительныйОтчет");
```

## Что именно считается

- Берутся только проведенные чеки ККМ и проведенные заказы клиентов.
- Статусы заказов, обеспечение, пробитость или архивность чеков отдельно не анализируются.
- Один и тот же товар из чеков и заказов попадает в одну строку отчета без разреза по характеристикам.
- Количество в этой строке - сумма количества из всех найденных строк чеков и заказов за период.
- Остаток в строке - учетный остаток минус количество из проведенных заказов за период.
